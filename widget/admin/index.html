Chế độ triển khai không cần Node

Dữ liệu chatbot được lưu trong file /widget-data/widget-data.json. Sau khi chỉnh sửa, hãy bấm Tải file deploy rồi upload đè file này lên host để áp dụng cho toàn bộ website.

Hướng dẫn nhanh: 1) Lưu tạm cấu hình. 2) Chọn 1 trong 2 cách deploy: tải file thủ công hoặc lưu trực tiếp lên host bằng khóa deploy.

Nút Tải file deploy: dùng khi bạn muốn backup/lưu file cục bộ. Nút Nạp file dữ liệu: dùng để khôi phục cấu hình từ một file đã tải trước đó.

Lưu trực tiếp lên host (không cần tải rồi upload)

Cần nhập khóa deploy để xác nhận quyền ghi file. Khóa mặc định là AhaDeploy@2026 (bạn có thể đổi trong file /widget/admin/deploy.aspx trên host).

Nút Kiểm tra kết nối deploy chỉ kiểm tra khóa và quyền ghi. Nút Lưu trực tiếp lên host sẽ ghi đè ngay file /widget-data/widget-data.json. 

Chưa kiểm tra deploy trực tiếp.

Lưu ý: nút Lưu tạm chỉ lưu vào trình duyệt hiện tại để bạn tiếp tục chỉnh nhanh. Muốn website thật cập nhật thì cần upload file deploy lên host.

Quản trị tài khoản (Super Admin)

Super Admin chỉ dùng khu vực này để quản lý trạng thái tài khoản. Tài khoản bị khóa sẽ không thể đăng nhập.

Thiết lập siêu nhanh (không cần code)

Chỉ cần điền các ô bên dưới. Hệ thống sẽ lưu cấu hình, tạo mã nhúng sẵn và đồng bộ về workspace hiện tại. Phần quét tự động đã tách riêng để bạn tập trung vào chất lượng câu trả lời.

Khối này không chạy quét tự động. Mục tiêu là hoàn thành cấu hình thật nhanh để bạn chuyển sang tab Nội dung và tinh chỉnh câu trả lời chuẩn theo ý.

Nút Sao chép mã nhúng: copy script để dán lên website. Nút Mở demo theo tài khoản này: mở tab xem thử đúng theo workspace đang thao tác. 

Sẵn sàng. Điền thông tin rồi bấm “Lưu thiết lập nhanh”.

Bạn đang chỉnh phần nâng cao. Bấm Lưu cài đặt nâng cao để lưu ngay mà không cần kéo xuống cuối trang.

Câu trả lời dự phòng

Cách bot dùng dữ liệu

  1. Ưu tiên 1: Bot dò trong Bộ câu trả lời mẫu theo mức độ gần nghĩa của câu hỏi.
  2. Ưu tiên 2: Nếu chưa khớp tốt, bot tìm trong Kho dữ liệu tài liệu và trích nội dung liên quan.
  3. Ưu tiên 3: Nếu vẫn không có kết quả, bot trả Câu trả lời dự phòng.

Mẹo tối ưu: mỗi ý chính nên có nhiều cách hỏi tương đương trong từ khóa để bot hiểu đúng ý hơn.

Kho dữ liệu tài liệu

Bổ sung dữ liệu liên tục cho khung chat. Hỗ trợ thêm tài liệu bằng văn bản hoặc tải file text (txt, md, csv, json, xlsx).

Tải file mẫu tài liệu CSV (Excel)

Nút Thêm tài liệu: thêm nội dung đang nhập tay. Nút Tải lên các file đã chọn: nhập hàng loạt từ file. Nút Tải lại dữ liệu: đọc lại danh sách tài liệu hiện có.

Nhập hàng loạt theo mẫu CSV/XLSX với cột: id,title,tags,text,sourceUrls. Nhiều giá trị trong tags hoặc sourceUrls thì ngăn cách bằng dấu |.

Công cụ quét tự động (tùy chọn)

Khối này dành cho trường hợp bạn muốn quét link nội bộ để tạo bản nháp. Với định hướng hiện tại, bạn có thể bỏ qua và tập trung biên tập nội dung thủ công để đạt độ chính xác cao nhất.

Cách dùng: 1. Lưu website. 2. Quét website và chờ trạng thái chuyển sang success. 3. Nạp bản nháp ở mục Duyệt nội dung sau quét, chỉnh tay và đánh dấu đã duyệt. 4. Khi đã duyệt đủ, mới đưa vào chatbot.

Ở bản không Node, trường header chỉ để lưu ghi chú cấu hình. Cách quét trang sau đăng nhập: đăng nhập sẵn bằng trình duyệt hiện tại trước khi bấm quét.

Nút Lưu website: lưu cấu hình nguồn quét. Nút Quét website và sinh hướng dẫn: bắt đầu crawl + sinh bản nháp. Nút Duyệt xong rồi đưa vào chatbot: chỉ dùng sau khi kiểm duyệt đầy đủ. 

Chưa có dữ liệu quét.

Duyệt nội dung sau quét

Đây là bước bắt buộc để kiểm tra lại toàn bộ câu trả lời tự sinh. Bạn có thể sửa tiêu đề, từ khóa, nội dung trả lời và nút gợi ý nhanh trước khi đưa lên chatbot.

Nút Đánh dấu duyệt tất cả: tick toàn bộ bản nháp. Nút Lưu bản nháp đã chỉnh: lưu lại nội dung đang sửa. Nút Xuất bản vào chatbot: đưa dữ liệu đã duyệt vào bot. 

Chưa nạp bản nháp cần duyệt.

Mỗi dòng gồm từ khóa -> câu trả lời + nút gợi ý nhanh.

Tải file mẫu CSV Tải file mẫu JSON

Nút Thêm dòng: thêm 1 hàng câu trả lời mới. Nút Nhập bộ câu trả lời từ file: nạp dữ liệu hàng loạt từ CSV/JSON.

Định dạng CSV/XLSX: id,keywords,answer,quickReplies. Trường keywords và quickReplies phân tách nhiều giá trị bằng dấu |.

Nút Tải lại từ file trên host: nạp dữ liệu đang chạy thật trên server. Nút Lưu tạm trên trình duyệt: chỉ lưu local để tiếp tục chỉnh nhanh.
